# AI Flow Testing

自动化流程驱动测试工具，通过解析自然语言场景描述自动生成和执行测试用例。

## 功能特点

- **自然语言场景描述**：用中文描述业务流程，自动生成测试用例
- **真实执行测试**：动态导入 Mock API，验证接口返回值和数据状态
- **智能字段映射**：自动分析 Vue 组件，提取字段信息，支持中文标签→英文字段名映射
- **跨模块测试**：支持跨模块的完整流程测试
- **测试报告生成**：自动生成详细的测试报告

## 快速开始

### 1. 编写模块文档

在 `docs/modules/` 目录下创建模块文档（如 `user-management.md`），包含：

1. **第0节：功能流程描述**（可选）

   ```markdown
   ## 0. 功能流程描述

   ### 场景名称：新增用户

   **涉及接口**：创建用户、查询列表

   **触发条件**：

   - 用户在用户管理页面
   - 点击"新增用户"按钮

   **操作流程**：

   1. 点击新增按钮
   2. 填写名称="张三"
   3. 填写邮箱="zhangsan@example.com"
   4. 点击保存按钮

   **完成标志**：列表中显示新增的用户
   ```

2. **第6节：测试用例**（必填）

   ```markdown
   ## 6. 测试用例（流程驱动）

   ### 6.1 流程测试

   #### 流程1：完整生命周期
   ```

   前置：无

   步骤：
   1. [新增] 创建用户，名称="测试用户"，邮箱="test@example.com" → 期望成功
   2. [查询] 查询该用户 → 期望存在
   3. [编辑] 修改名称为"测试用户V2" → 期望成功
   4. [验证] 查询该用户 → 期望名称已更新
   5. [删除] 删除该用户 → 期望成功
   6. [验证] 查询该用户 → 期望不存在

   最终验证：用户已被完全清除

   ```

   ```

3. **第4.1节：接口实现清单**（必填）

   ```markdown
   ## 4.1 接口实现清单（必填）

   | 组件调用方法      | Mock 实现方法 | 映射验证 | 备注 |
   | ----------------- | ------------- | :------: | ---- |
   | userApi.getList() | getList       |    ✅    |      |
   | userApi.create()  | create        |    ✅    |      |
   | userApi.update()  | update        |    ✅    |      |
   | userApi.remove()  | remove        |    ✅    |      |
   ```

### 2. 运行测试

```bash
# 测试单个模块
node scripts/ai-test.js --module=user-management

# 测试指定流程
node scripts/ai-test.js --module=user-management --flow=1

# 测试所有模块
node scripts/ai-test.js --all
```

### 3. 查看测试报告

测试报告会保存在 `docs/modules/{module-name}.test-report.md`

## 场景描述格式规范

### 操作类型关键词

| 关键词 | 说明          | 示例              |
| ------ | ------------- | ----------------- |
| 打开   | 进入页面/弹窗 | 打开新增弹窗      |
| 填写   | 输入数据      | 填写名称="张三"   |
| 选择   | 从列表中选择  | 选择类型="管理员" |
| 点击   | 触发按钮操作  | 点击保存按钮      |
| 调用   | 后端接口      | 调用创建接口      |
| 返回   | 接口响应      | 返回成功          |
| 刷新   | 更新页面数据  | 刷新列表          |

### 参数格式

- **正确**：`名称="测试数据"` 或 `name="test data"`
- **错误**：`名称为测试数据`（无法解析）

### 测试用例步骤格式

每步必须包含操作类型和期望结果：

```
  1. [新增] 创建用户，名称="张三" → 期望成功
  2. [查询] 查询该用户 → 期望存在
  3. [删除] 删除该用户，id=99999 → 期望失败，提示用户不存在
```

## Mock API 要求

Mock API 模块必须导出以下方法：

```javascript
export const xxxApi = {
  async getList(params) { ... },      // 查询列表
  async create(data) { ... },          // 新增
  async update(id, data) { ... },      // 编辑
  async remove(id) { ... },            // 删除
  async on(id) { ... },                // 启用（可选）
  async off(id) { ... },               // 禁用（可选）
}
```

### 返回值格式规范

**标准返回格式**：

```javascript
// 成功
{ code: 200, data: {...}, msg: '操作成功' }

// 失败
{ code: 400, data: null, msg: '错误信息' }
```

**分页数据格式**：

```javascript
{
  code: 200,
  data: {
    records: [...],   // 数据列表（必填）
    total: 100,       // 总记录数（必填）
    page: 1,          // 当前页码（必填）
    size: 10          // 每页大小（必填）
  },
  msg: '查询成功'
}
```

### 状态码规范

| Code | 说明         | 使用场景                     |
| ---- | ------------ | ---------------------------- |
| 200  | 操作成功     | 所有成功操作                 |
| 400  | 请求参数错误 | 参数缺失、格式错误、校验失败 |
| 401  | 未授权       | 需要登录才能访问             |
| 403  | 禁止访问     | 权限不足                     |
| 404  | 资源不存在   | 数据不存在                   |
| 409  | 资源冲突     | 数据重复、状态冲突           |
| 422  | 数据验证失败 | 业务规则验证失败             |
| 500  | 服务器错误   | 未知错误、系统异常           |

### 错误提示规范

**好的错误提示**：

- ✅ "优惠券名称不能为空"
- ✅ "使用门槛不能小于面额"
- ✅ "优惠券名称已存在"

**不好的错误提示**：

- ❌ "参数错误"（不明确）
- ❌ "操作失败"（无意义）

> 📋 **详细的 Mock API 规范请参考**：`docs/MOCK_API_SPEC.md`

## 故障排查

### 问题1：场景描述未生成测试用例

**原因**：

- 场景描述格式不正确
- 缺少 `### 场景名称：` 标记

**解决**：参考模板文件 `template/MODULE-TEMPLATE.md`

### 问题2：接口映射验证失败

**原因**：

- Mock API 中缺少对应方法
- 方法名拼写错误

**解决**：

1. 检查 Mock API 是否导出了所有必需方法
2. 检查接口实现清单中的方法名是否正确

### 问题3：参数提取失败

**原因**：

- 场景描述中的参数格式不正确
- 字段映射缺失

**解决**：

1. 使用 `字段名="值"` 或 `字段名=值` 格式
2. 确保 Vue 组件中的字段有正确的 label 属性
