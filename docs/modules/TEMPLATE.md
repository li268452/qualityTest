## 6. 测试用例（流程驱动）

> 测试思路：按业务场景走流程，每步只判断成功/失败，
> 上一步的结果作为下一步的输入，失败时才分析具体原因。

### 6.1 流程测试

#### 流程1：（例：字典管理完整生命周期）

```
前置：无

步骤：
  1. [新增] 创建字典类型 → 期望成功，记录类型ID
  2. [查询] 查询该类型 → 期望存在
  3. [新增] 在该类型下创建字典数据A → 期望成功
  4. [新增] 在该类型下创建字典数据B → 期望成功
  5. [编辑] 修改数据A的排序号 → 期望成功
  6. [验证] 查询该类型数据列表 → 期望按排序号排列
  7. [删除] 删除数据B → 期望成功
  8. [验证] 查询列表 → 期望数据B不存在
  9. [删除] 删除字典类型 → 期望成功
  10. [验证] 查询该类型 → 期望不存在

最终验证：该类型及所有数据均已清除
```

#### 流程2：（例：异常拦截流程）

```
前置：已存在字典类型"user_status"

步骤：
  1. [新增] 使用重复编码创建 → 期望失败，提示编码已存在
  2. [新增] 使用空名称创建 → 期望失败，提示名称不能为空
  3. [删除] 删除被引用的字典类型 → 期望失败，提示已被使用
  4. [验证] 原数据未被影响
```
