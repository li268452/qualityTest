# AI Flow Testing Skill

自动化流程驱动测试工具 - 让测试用例编写变得简单自然。

## 简介

AI Flow Testing 是一个创新的测试工具，它允许你用**自然语言**描述业务流程，然后自动生成并执行测试用例。

### 核心优势

- ✅ **自然语言编写**：用中文描述业务流程，无需编写测试代码
- ✅ **真实执行测试**：真正调用 Mock API，验证接口和数据状态
- ✅ **智能字段映射**：自动分析 Vue 组件，支持中文标签
- ✅ **测试报告自动生成**：详细的测试报告，包含失败原因和修复建议

## 快速开始

### 第一步：编写模块文档

在 `docs/modules/` 目录下创建模块文档（可参考 `template/MODULE-TEMPLATE.md`）：

```markdown
# 用户管理模块

## 0. 功能流程描述

### 场景名称：新增用户

**操作流程**：

1. 点击新增按钮
2. 填写名称="张三"
3. 填写邮箱="zhangsan@example.com"
4. 点击保存按钮

**完成标志**：列表中显示新增的用户

---
```

### 第二步：运行测试

```bash
# 测试单个模块
node .claude/skills/ai-flow-testing/scripts/ai-test.js --module=user-management

# 测试所有模块
node .claude/skills/ai-flow-testing/scripts/ai-test.js --all
```

### 第三步：查看报告

测试报告保存在 `docs/modules/user-management.test-report.md`

## 测试用例格式

### 正常流程

```
前置：无

步骤：
  1. [新增] 创建用户，名称="张三"，邮箱="zhangsan@example.com" → 期望成功
  2. [查询] 查询该用户 → 期望存在
  3. [编辑] 修改名称为"张三V2" → 期望成功
  4. [验证] 查询该用户 → 期望名称已更新
  5. [删除] 删除该用户 → 期望成功
  6. [验证] 查询该用户 → 期望不存在

最终验证：用户已被完全清除
```

### 异常流程

```
前置：无

步骤：
  1. [新增] 创建用户，名称="" → 期望失败，提示名称不能为空
  2. [新增] 创建用户，名称="重复名称" → 期望失败，提示名称已存在
  3. [新增] 创建用户，名称="测试"，邮箱="invalid" → 期望失败，提示邮箱格式不正确

最终验证：所有异常被正确拦截
```

### 跨模块流程

```
前置：字典管理模块和用户管理模块均已启动

步骤：
  1. [切换] 切换到字典管理页面 (UI操作，API测试跳过)
  2. [选择] 选择"用户状态"字典 (UI操作，API测试跳过)
  3. [新增] 新增字典数据，标签="冻结"，值="frozen"，排序=3 → 期望成功
  4. [验证] 查询"用户状态" → 期望包含"冻结"
  5. [切换] 切换到用户管理页面 (UI操作，API测试跳过)
  6. [新增] 创建用户，名称="测试用户"，状态="冻结" → 期望成功

最终验证：从字典新增类型到用户使用的完整链路打通
```

## 参数格式规范

### 正确格式 ✅

- `名称="张三"` - 带引号的字符串
- `age=25` - 数字
- `enabled=true` - 布尔值
- `标签="正常", 值="1", 排序=1` - 多个参数

### 错误格式 ❌

- `名称为张三` - 无法解析
- `名称：张三` - 应使用等号
- `名称、邮箱、年龄` - 缺少值

## 常见问题

### Q1: 场景描述没有生成测试用例？

**A:** 检查以下几点：

1. 确保使用了正确的标记 `### 场景名称：`
2. 确保场景描述在 `## 0. 功能流程描述` 章节
3. 检查操作流程格式是否正确

### Q2: 接口映射验证失败？

**A:** 检查 Mock API 是否导出了对应方法：

```javascript
export const userApi = {
  async getList(params) { ... },
  async create(data) { ... },
  async update(id, data) { ... },
  async remove(id) { ... },
}
```

### Q3: 参数没有被正确提取？

**A:** 检查参数格式：

- 使用 `字段名="值"` 或 `字段名=值` 格式
- 确保字段名在 Vue 组件中有对应的 label 属性

## 项目集成

AI Flow Testing 支持多种 Vue3 项目结构，会自动适配。

### 推荐结构

```
your-project/
├── src/
│   ├── views/              # 页面组件（或 pages/）
│   │   └── user/
│   │       ├── UserManagement.vue
│   │       └── UserForm.vue
│   ├── api/                # API 请求
│   │   └── user.js         # 用户 API（包含 mockApi）
│   └── main.js
├── docs/                   # 模块文档（可选）
│   └── user-management.md
└── package.json
```

### 支持的结构

**标准 Vue3 结构**：

```
src/
├── views/           # 页面
├── components/      # 组件
├── api/             # API
└── mock/            # Mock（可选）
```

**模块化结构**：

```
src/
└── modules/
    └── user/
        ├── components/
        ├── api/
        └── mock/
```

### 文档位置（可选）

如果使用独立文档，可放在：

- `docs/user-management.md`
- `docs/modules/user-management.md`
- `test/docs/user-management.md`

如果不使用文档，AI 会从代码注释中提取信息。

### package.json 配置

确保 `package.json` 包含：

```json
{
  "type": "module"
}
```

### Mock API 返回格式

确保 Mock API 返回正确的格式：

**标准格式**：

```javascript
// 成功
{ code: 200, data: {...}, msg: '操作成功' }

// 失败
{ code: 400, data: null, msg: '错误信息' }
```

**分页数据**：

```javascript
{
  code: 200,
  data: {
    records: [...],  // 数据列表
    total: 100,      // 总记录数
    page: 1,         // 当前页码
    size: 10         // 每页大小
  },
  msg: '查询成功'
}
```

**状态码**：

- `200` - 操作成功
- `400` - 请求参数错误
- `404` - 资源不存在
- `409` - 资源冲突（如数据重复）
- `422` - 业务规则验证失败

> 📋 **详细的 Mock API 规范请参考**：`docs/MOCK_API_SPEC.md`
